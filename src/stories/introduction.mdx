# Credo React Component Library

Welcome to the **Credo React Component Library** documentation! This comprehensive design system provides accessible, themeable, and performant React components built with Tailwind CSS v4.

## üöÄ Quick Start

### Installation

```bash
npm install @credo-labs/react
```

### Setup

Import the library styles in your application:

```css
/* In your main CSS file */
@import "@credo-labs/react/styles.css";
@import "tailwindcss";
```

### Basic Usage

```tsx
import { Button } from "@credo-labs/react/ui/button";
import { CredoLogo } from "@credo-labs/react/ui/icons";

function App() {
  return (
    <div className="p-6">
      <div className="flex items-center gap-2 mb-4">
        <CredoLogo className="w-8 h-8" />
        <h1 className="text-display-sm font-bold text-gray-900">
          Welcome to Credo
        </h1>
      </div>

      <div className="flex gap-3">
        <Button variant="primary">Get Started</Button>
        <Button variant="outline">Learn More</Button>
      </div>
    </div>
  );
}
```

## üé® Design Principles

### Accessibility First

Every component is built with accessibility in mind, following WCAG 2.1 guidelines:

- Semantic HTML structure
- Keyboard navigation support
- Screen reader compatibility
- Proper focus management
- High contrast ratios

### Performance Optimized

- **Tree-shakable**: Import only what you need
- **Lightweight**: Minimal runtime overhead
- **Modern**: Built for React 18+ and modern browsers

### Customizable

- **CSS Custom Properties**: Easy theme customization
- **Tailwind Integration**: Use utility classes for styling
- **Flexible APIs**: Comprehensive prop interfaces

## üìã Import Strategies

### Recommended: Granular Imports

For optimal bundle size and tree-shaking:

```tsx
// ‚úÖ Best for performance
import { Button } from "@credo-labs/react/ui/button";
import { Card, CardHeader, CardContent } from "@credo-labs/react/ui/card";
import { CredoLogo, ArrowRight } from "@credo-labs/react/ui/icons";
```

### Alternative: Barrel Imports

For convenience (less optimal for tree-shaking):

```tsx
// ‚úÖ Convenient but larger bundle
import { Button, Card } from "@credo-labs/react";
// Icons should always use specific imports
import { CredoLogo } from "@credo-labs/react/ui/icons";
```

## üé® Theming Guide

### Basic Theme Customization

Override CSS custom properties to customize the theme:

```css
:root {
  /* Primary brand color */
  --primary-500: #3b82f6; /* Blue instead of orange */

  /* Border radius */
  --radius: 12px; /* More rounded */

  /* Typography */
  --font-sans: Inter, ui-sans-serif, system-ui, sans-serif;
}
```

### Advanced Theming

Create multiple themes using CSS classes:

```css
/* Light theme (default) */
.theme-light {
  --color-background: #ffffff;
  --color-foreground: #1f2937;
}

/* Dark theme */
.theme-dark {
  --color-background: #1f2937;
  --color-foreground: #f9fafb;
}

/* Brand theme */
.theme-brand {
  --primary-500: #8b5cf6; /* Purple brand */
  --color-background: #faf5ff;
}
```

Apply themes dynamically:

```tsx
function App() {
  const [theme, setTheme] = useState("light");

  return <div className={`theme-${theme}`}>{/* Your app content */}</div>;
}
```

## üéØ Best Practices

### Performance

- Use granular imports for better tree-shaking
- Import styles only once in your application root
- Leverage React.memo for expensive components

### Accessibility

- Always provide meaningful labels for form inputs
- Use semantic HTML elements
- Test with keyboard navigation
- Verify screen reader compatibility

### Styling

- Use Tailwind utilities for customization
- Follow the established design token system
- Avoid overriding component internals
- Prefer theme customization over component-specific styles
